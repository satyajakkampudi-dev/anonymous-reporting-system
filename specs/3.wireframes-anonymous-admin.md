# Wireframes — anonymous-admin (per-platform, low-fidelity)

> LoG.ai Layer 3 Deliverable 2. **No Figma** (PM choice, as for anonymous-user) — structure-only,
> UX-analysed wireframes grounded in real-world triage/case-management patterns (Linear/Zendesk queue
> + ticket detail; NHS/Speak-Up compliance dashboards for anonymity-safe analytics). **Mobile and web
> get separate, purpose-built layouts** (REQUIREMENTS §9.1) — not one responsive layout. Each screen
> maps to a `renderWeb` / `renderMobile` pair under `src/sections/display/<screen>/`. Theme tokens
> shared via `lib/utils/theme.js`. `[RO]` = read-only · dashed = conditional.
>
> **Anonymity is the dominant constraint:** every admin screen below shows **zero reporter identity**
> — no name, email, phone, cabin, or userId anywhere. The admin sees the `adminProjection` set only.
>
> Markers: section boundaries `┌─┐`, status pill `‹ OPEN ›`, severity tone `«CRITICAL»`, button
> `[ Label ]`, badge `⚑`.

## 6. On-call availability (A-F20) — `custom_card`

**UX intent:** one-tap presence. Three mutually-exclusive states (not a boolean toggle).

### Web / Mobile
```

│ On-call status                 │
│ Current: ‹ AVAILABLE ›         │
│ [ Available ] [ Busy ]         │
```

Only `available` admins are rung (A-F21). Answering a call auto-sets `busy`.

## 7. Incoming call (A-F21/F22) — `custom_card`, action-driven

**UX intent:** answer fast; reveal nothing about the caller. Banner is generic; **no name/id/email**
(masked, ER-A5). First admin to answer wins (atomic claim); the rest stop ringing.

### Mobile / Web (ring card)
```
┌───────────────────────────────┐
│ 📞  Incoming anonymous call    │  ← no caller identity, ever
│                                │
│        [  Answer  ]            │  → atomic claim → ACTIVE, you = busy
│        [  Dismiss ]            │  → others keep ringing
└───────────────────────────────┘
In call → voice only, video off, NOT recorded (ER-A5).
Hang-up / drop → ENDED (duration recorded); mid-call inactivity → ENDED.
```

## Cross-cutting render notes (§9.1)

- Each screen above is a `src/sections/display/<screen>/{index,web,mobile}.js` triple. `index.js`
  reads prepared data from a `state.setField` stash and calls
  `renderForPlatform(data, { web: renderWeb, mobile: renderMobile })` — one dispatch, no inline
  `isMobile()` branching.
- Status pill labels/tones come from `lib/ticket-status.js` `STATUS_META`; severity tones and the
  priority badge from `lib/utils/theme.js`. `escapeHtml` + date/relative-date builders from
  `lib/utils/format.js`.
- Evidence links are **signed server-side before `sendResponse()`** (A-F7) — never the raw S3 key,
  never signed inside `onResponse`.
- No screen binds or renders a reporter-identity field (C1, ER-A2/A3).
