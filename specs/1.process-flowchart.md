# Process Flowchart - Anonymous Reporting System

> LoG.ai Layer 1. Swimlanes = actors + the two micro-apps + shared data + the job scheduler.
> Handoffs cross app boundaries via bot-to-bot `MSG_*`; **no payload carries reporter identity**.
> Source: [`../REQUIREMENTS.md`](../REQUIREMENTS.md) §7 flows, decisions D1–D17.

```mermaid
sequenceDiagram
    actor R as Reporter (crew / shore)
    participant U as anonymous-user (bot)
    participant DB as reports / call-queue / admin-users<br/>(shared, *_${systemId})
    participant A as anonymous-admin (bot)
    actor P as Primary admin
    actor S as Secondary admin
    participant J as System (job scheduler)

    Note over R,S: All cross-app payloads are reporter-identity-free.

    %% --- Submission ---
    rect rgb(235,245,255)
    Note over R,A: 1. Report submission (trigger: reporter submits)
    R->>U: Fill report form + evidence
    U->>U: validate + sanitise + generate reportId (lib)
    U->>U: resolveAssignees(report): againstAdmin ? SECONDARY : PRIMARY (scope GLOBAL, D17)
    U->>DB: save report (status OPEN)
    U-->>R: tracking ID + confirmation (email + web push)
    U->>A: MSG_NEW_REPORT {reportId, category, urgency, severity, assignedTo, createdOn}
    A->>P: notify assigned admins (email + push)
    A->>J: schedule auto-escalate (CRITICAL +1d / others +3d, D2)
    end

    %% --- Tracking ---
    rect rgb(240,255,240)
    Note over R,DB: 2. Tracking (trigger: reporter opens My Reports)
    R->>U: open My Reports / a report
    U->>DB: query { reporterId: state.user.userId }
    U->>U: assert report.reporterId === userId (ownership)
    U-->>R: card list + detail + status timeline
    end

    %% --- Admin triage ---
    rect rgb(255,250,235)
    Note over P,DB: 3. Admin triage -> resolve / escalate
    P->>A: open queue (role-filtered, identity-stripped via adminProjection)
    A->>DB: loadReportsForAdmin() - projection always applied (ER-A3)
    P->>A: take review (OPEN -> UNDER_REVIEW, validated + audited)
    alt Resolve
        P->>A: enter resolution -> RESOLVED
        A->>DB: status -> RESOLVED (audited)
        A->>U: MSG_REPORT_RESOLVED {reportId, resolvedOn}
        U-->>R: notify reporter (email + push)
        A->>J: schedule auto-close (+30d, D2)
    else Escalate (Admin -> Admin handoff)
        P->>A: escalate
        A->>DB: status -> ESCALATED, assignedTo = SECONDARY (audited)
        A->>S: notify secondary admins (email + push)
        S->>A: take review (ESCALATED -> UNDER_REVIEW) -> resolve
    end
    end

    %% --- Accept / reject / withdraw ---
    rect rgb(245,240,255)
    Note over R,A: 4. Reporter accept / reject / withdraw
    alt Accept
        R->>U: accept resolution
        U->>DB: RESOLVED -> CLOSED_BY_USER (terminal)
    else Reject (once only, D10)
        R->>U: reject (reason)
        U->>DB: RESOLVED -> REOPENED, reopenCount 0->1
        U->>A: MSG_REPORT_REOPENED {reportId, reopenCount, rejectReason}
        A->>P: notify assigned admins
    else Withdraw (OPEN / UNDER_REVIEW, D16)
        R->>U: withdraw
        U->>DB: status -> WITHDRAWN (terminal)
    end
    end

    %% --- Automation ---
    rect rgb(255,240,240)
    Note over J,DB: 5. Automation (non-user triggers; idempotent ER-B8)
    J->>A: fire auto-escalate (reportId)
    A->>DB: if still OPEN/unactioned -> ESCALATED, assignedTo = SECONDARY
    A->>S: notify secondary (else no-op + log)
    J->>A: fire auto-close (reportId)
    A->>DB: if still RESOLVED -> CLOSED_BY_SYSTEM
    A->>U: MSG_REPORT_CLOSED {reportId, closeType}
    U-->>R: notify reporter (else no-op + log)
    J->>A: SLA backstop (OPEN 24h / ESCALATED 24h, D11) -> digest to all admins
    end

    %% --- Anonymous call ---
    rect rgb(235,255,255)
    Note over R,S: 6. Anonymous voice call (+ voicemail fallback, D7/D8/D12)
    R->>U: tap "Call compliance (anonymous)"
    U->>U: create voice-only meeting (masked host, allowGuests, video muted)
    U->>U: reporter joins as masked guest ("Anonymous Reporter")
    U->>DB: create call-queue entry (RINGING, identity-free)
    U->>DB: load admin-users where availability = available
    U->>A: MSG_INCOMING_CALL {callRef, meetingId} + VoIP push (identity-free)
    alt An admin answers (atomic claim)
        A->>DB: entry -> ACTIVE, attendedBy = admin; admin -> busy
        A->>A: MSG_CALL_STOP_RING {callRef, meetingId} to other admins
        Note over R,A: voice call - reporter stays anonymous (no recording, ER-A5)
        A->>DB: on hang-up -> ENDED (duration recorded)
    else No available admin / 30s timeout (D7)
        U->>DB: entry -> MISSED
        U-->>R: "No one available - leave a voice message"
        R->>U: record voicemail (<=3 min / 25 MB, D7) -> domain S3
        U->>U: auto-create report (source = CALL, voicemail as evidence, status OPEN)
        U->>A: MSG_NEW_REPORT (identity-free)
    end
    end
```
