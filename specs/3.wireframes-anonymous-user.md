# Wireframes — anonymous-user (per-platform, low-fidelity)

> LoG.ai Layer 3 Deliverable 2. **No Figma** (PM choice) — these are structure-only, UX-analysed
> wireframes grounded in real-world reporting/whistleblowing + ticketing patterns (NHS/Speak-Up
> portals for anonymity-first framing; Linear/Zendesk for ticket detail + timeline). **Mobile and web
> get separate, purpose-built layouts** (REQUIREMENTS §9.1) — not one responsive layout. Each screen
> maps to a `renderWeb` / `renderMobile` pair under `src/sections/display/<screen>/`. Theme tokens
> shared via `lib/utils/theme.js`. `[RO]` = read-only · dashed = conditional.
>
> Markers: section boundaries `┌─┐`, status pill `‹ OPEN ›`, button `[ Label ]`.

---

## 2. Submit a report (U-F3/4/5/6/7) — `standard` Form Doc + pre-submit guard

**UX intent:** a long form on a slow maritime link is risky — so (a) **draft autosave** is signalled,
(b) the **anonymity guard** (U-F5) sits *inline at the top* and again at submit, (c) conditional
contact field appears only when needed, (d) evidence shows count/limit. Mobile = single column,
collapsible groups; web = two columns, all groups open.

## 5. Call compliance (anonymous) (U-F15/16/17) — action + voicemail

**UX intent:** mirror a phone-call screen but make the *anonymity* explicit ("You will appear as
Anonymous Reporter; this call is not recorded"). Clear fallback to voicemail on no-answer.

### Mobile (web is the same flow, wider)
```
┌───────────────────────────────┐        ┌───────────────────────────────┐
│   📞 Calling compliance…       │        │   No one is available right    │
│                                │        │   now.                         │
│  You appear as                 │  30s   │                                │
│  "Anonymous Reporter".         │ ─────▶ │  Leave a voicemail (≤3 min)?   │
│  Voice only · not recorded.    │ no ans │  ┌─────────────────────────┐   │
│                                │        │  │  ● Record   (●REC 0:00)  │   │
│        ◍ ringing…              │        │  └─────────────────────────┘   │
│                                │        │  [ Send voicemail ] [ Cancel ] │
│       [  Hang up  ]            │        │  → auto-creates an OPEN report │
└───────────────────────────────┘        └───────────────────────────────┘
   Hang up before answer → ABANDONED (no report).
   Answered → in-call screen (mute/hang-up); drop → ENDED via inactivity timeout.
   Voicemail sent → "Report RPT-… created from your voicemail." (source=CALL)
```

## Cross-cutting UX notes
- **Status pills** use one tone map (`lib/ticket-status.js` meta → `lib/utils/theme.js`) so the same
  status reads identically on mobile, web, and the admin app.
- **Anonymity reassurance** recurs at every identity-risk moment: home banner, submit guard, call
  masking notice (ER-D16 trust UX).
- **Slow-link resilience:** draft autosave on submit; signed-URL re-fetch on expired evidence links;
  best-effort notifications never block the UI.
- **Empty/error states are first-class:** no-reports, not-your-report, validation failures, no-admin
  voicemail fallback — each has explicit copy above.
