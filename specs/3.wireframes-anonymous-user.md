# Wireframes - anonymous-user (per-platform, low-fidelity)

> LoG.ai Layer 3 Deliverable 2. **No Figma** (PM choice) - these are structure-only, UX-analysed
> wireframes grounded in real-world reporting/whistleblowing + ticketing patterns (NHS/Speak-Up
> portals for anonymity-first framing; Linear/Zendesk for ticket detail + timeline). **Mobile and web
> get separate, purpose-built layouts** (REQUIREMENTS В§9.1) - not one responsive layout. Each screen
> maps to a `renderWeb` / `renderMobile` pair under `src/sections/display/<screen>/`. Theme tokens
> shared via `lib/utils/theme.js`. `[RO]` = read-only В· dashed = conditional.
>
> Markers: section boundaries `в”Ңв”Җв”җ`, status pill `вҖ№ OPEN вҖә`, button `[ Label ]`.

## 0. Access gate - Restricted (U-F0a) - standalone refusal card

**UX intent:** a user without the reporter licence (`quitelineenduser`) must hit a clear, calm wall
- never a half-rendered Home screen. Static copy, single centred card, no actions (there is nowhere
in-app to navigate to; the licence is redeemed in the FrontM profile). Sent on DENY by the access
gate BEFORE any bootstrap; identical copy on mobile/web (web centres a narrow column).

## Cross-cutting UX notes
- **Status pills** use one tone map (`lib/ticket-status.js` meta вҶ’ `lib/utils/theme.js`) so the same
  status reads identically on mobile, web, and the admin app.
- **Anonymity reassurance** recurs at every identity-risk moment: home banner, submit guard, call
  masking notice (ER-D16 trust UX).
- **Slow-link resilience:** draft autosave on submit; signed-URL re-fetch on expired evidence links;
  best-effort notifications never block the UI.
- **Empty/error states are first-class:** no-reports, not-your-report, validation failures, no-admin
  voicemail fallback - each has explicit copy above.
